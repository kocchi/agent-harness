# 場（Place）テンプレート

場ごとの実行構造を定義する。メインエージェントはオーケストレーターとして、場のテンプレートに従ってサブエージェントに委譲する。

## 運用原則

- **メイン**: 憑依なし。委譲のみ。
- **サブエージェント**: 複数人格を憑依させ、議論させる。
- **議論の過程**: 必ず記録する（人間の学習用）。

## テンプレート構造

各場は以下のステップを持つ:

1. **分析**: 対象を分析し、ペルソナ選定の変数を得る
2. **召喚条件生成**: 変数から「どのような人物を召喚するか」の条件を生成（固定リストは使わない）
3. **議論**: 複数人格が議論（persona-council サブエージェントに委譲）
4. **統合**: 議論結果を統合

## 出力形式

```yaml
output:
  discussion:
    - speaker: "人格名"
      statement: "発言内容"
      rationale: "根拠"
      response_to: "誰への応答か（あれば）"
  integration:
    stop_doing: []
    start_doing: []
    preserve: []
  artifacts:
    discussion_log: "path/to/discussion.md"
```
