# セッション初期化

このリポジトリで作業開始時に実行。

## 0. ルール検査（session_start）

セッション開始時に `rules-validator` を呼び出してルール検査を実行:

```
rules-validator を timing: session_start で呼び出してください。
```

検査内容:
- 前セッションで source/ と generated/ の不整合はなかったか？
- 週次リサーチが更新されているか？

## 1. 週次リサーチチェック

`research/weekly/` の最新ファイルを確認:
- 1週間以上経過 → `weekly-research` サブエージェントを呼び出す
- 1週間未満 → スキップ

## 2. 確認事項

### SSoT運用
- `source/` を編集 → `./scripts/generate.sh` → コミット
- 自動生成ファイルは直接編集しない

### 基本原理
- P1: コンテキスト競合の排除
- P2: 中間生成物で合意を運ぶ
- P3: 道具を減らす
- P4: 知識は資産として蓄積
- P5: 検査可能性を保つ

### 文書規約
- `__強調__` を使う（`**` は使わない）

## 3. 中間生成物
- __契約__: 何を作るか
- __失敗モード__: 何が壊れうるか
- __テスト戦略__: どう検証するか
